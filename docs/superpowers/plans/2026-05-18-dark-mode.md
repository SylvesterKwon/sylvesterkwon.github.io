# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dark mode support to the Gatsby blog that activates automatically based on the user's OS preference (`prefers-color-scheme: dark`).

**Architecture:** Pure CSS approach — no JS state, no toggle UI, no localStorage. Override CSS custom properties inside a `@media (prefers-color-scheme: dark)` block. Swap Prism syntax highlighting theme (light → Tomorrow) and Utterances comment theme via cascade and native config. Add `<meta name="color-scheme">` so the browser knows we support both.

**Tech Stack:** Gatsby 5, React 18, vanilla CSS with custom properties, PrismJS (syntax highlighting), Utterances (GitHub-issues-based comments).

**Spec:** `docs/superpowers/specs/2026-05-18-dark-mode-design.md`

**Project note on testing:** This project has no unit test framework (`package.json` `test` script is `echo "Write tests! ..." && exit 1`). Verification is done by running `npm run develop` and visually inspecting pages while toggling the OS theme. Each task includes a manual verification step.

## Task 2: Create `src/prism-dark.css` (Tomorrow theme wrapped in dark media query)

**Files:**
- Create: `src/prism-dark.css`
- Modify: `gatsby-browser.js` (add one import line)

- [ ] **Step 1: Create `src/prism-dark.css` with the full content below**

This is PrismJS's canonical Tomorrow theme (MIT-licensed, originally based on the Tomorrow color palette by Chris Kempson), with every rule wrapped in a `@media (prefers-color-scheme: dark)` block so it only activates in dark mode. In light mode, the existing `prismjs/themes/prism.css` continues to apply.

```css
/**
 * prism-dark.css — Tomorrow theme, scoped to dark mode only.
 * Based on PrismJS's bundled prism-tomorrow.css (MIT).
 * Activates only when the user's OS is in dark mode; otherwise the default
 * Prism light theme (imported in gatsby-browser.js) remains in effect.
 */

@media (prefers-color-scheme: dark) {
  code[class*="language-"],
  pre[class*="language-"] {
    color: #ccc;
    background: none;
    font-family: Consolas, Monaco, "Andale Mono", "Ubuntu Mono", monospace;
    font-size: 1em;
    text-align: left;
    white-space: pre;
    word-spacing: normal;
    word-break: normal;
    word-wrap: normal;
    line-height: 1.5;

    -moz-tab-size: 4;
    -o-tab-size: 4;
    tab-size: 4;

    -webkit-hyphens: none;
    -moz-hyphens: none;
    -ms-hyphens: none;
    hyphens: none;
  }

  /* Code blocks */
  pre[class*="language-"] {
    padding: 1em;
    margin: 0.5em 0;
    overflow: auto;
  }

  :not(pre) > code[class*="language-"],
  pre[class*="language-"] {
    background: #2d2d2d;
  }

  /* Inline code */
  :not(pre) > code[class*="language-"] {
    padding: 0.1em;
    border-radius: 0.3em;
    white-space: normal;
  }

  .token.comment,
  .token.block-comment,
  .token.prolog,
  .token.doctype,
  .token.cdata {
    color: #999;
  }

  .token.punctuation {
    color: #ccc;
  }

  .token.tag,
  .token.attr-name,
  .token.namespace,
  .token.deleted {
    color: #e2777a;
  }

  .token.function-name {
    color: #6196cc;
  }

  .token.boolean,
  .token.number,
  .token.function {
    color: #f08d49;
  }

  .token.property,

  .token.constant,

  }

  .token.selector,
  .token.important,

  .token.builtin {
    color: #cc99cd;
  }

  .token.string,
  .token.char,
  .token.attr-value,
  .token.regex,

  }

  .token.operator,
```

## Task 4: Add `<meta name="color-scheme">` to `gatsby-ssr.js`

**Files:**
- Modify: `gatsby-ssr.js`

- [ ] **Step 1: Replace the contents of `gatsby-ssr.js`**

Current file content:

```js
/**
 * Implement Gatsby's SSR (Server Side Rendering) APIs in this file.
 *
 * See: https://www.gatsbyjs.com/docs/reference/config-files/gatsby-ssr/
 */

/**
 * @type {import('gatsby').GatsbySSR['onRenderBody']}
 */
exports.onRenderBody = ({ setHtmlAttributes }) => {
  setHtmlAttributes({ lang: `en` })
}
```

Replace with:

```js
/**
 * Implement Gatsby's SSR (Server Side Rendering) APIs in this file.
 *
 * See: https://www.gatsbyjs.com/docs/reference/config-files/gatsby-ssr/
 */

const React = require("react")

/**
 * @type {import('gatsby').GatsbySSR['onRenderBody']}
 */
exports.onRenderBody = ({ setHtmlAttributes, setHeadComponents }) => {
  setHtmlAttributes({ lang: `en` })
  setHeadComponents([
    React.createElement("meta", {
      key: "color-scheme",
      name: "color-scheme",
      content: "light dark",
    }),
  ])
}
```

**Why `React.createElement` instead of JSX:** `gatsby-ssr.js` is a CommonJS file (`exports.onRenderBody = ...`) and Gatsby's default Babel config for these top-level config files may not handle JSX. Using `React.createElement` directly avoids that concern.

- [ ] **Step 2: Restart dev server and verify the meta tag**

Stop and restart `npm run develop`.

Open any page in the browser, view source (or use DevTools → Elements), and confirm `<meta name="color-scheme" content="light dark">` appears in the `<head>`.

Also do a hard reload (Cmd+Shift+R) of a page while in OS dark mode: the page should render with a dark background immediately (no white flash), because the browser knows the page supports dark mode before any CSS has even parsed.

- [ ] **Step 3: Commit**

```bash

git commit -m "feat(ssr): declare color-scheme support for light and dark"
```

- Background too dark / too warm: tweak `--color-bg` (try `#1a1d23`, `#22272e`, or `#0d1117` for a more classic GitHub Dark feel).
- Text too dim / too bright: tweak `--color-text` (try `#d4d4dc` for brighter, `#b3b9c4` for softer).
- Link blue too saturated: try `#79b8ff` or `#69b1ff`.

Each tuning change gets its own commit, e.g.:

